Checks that require CI / a real browser CI ONLY

Console errors & runtime JS failures and Lighthouse / Core Web Vitals cannot be measured by static or same-origin fetch analysis. They are produced by the GitHub Actions workflow (Playwright + Lighthouse jobs) and uploaded as build artifacts. HTTP/security headers, redirects and TLS are enforced at the host/Cloudflare layer and are not visible from repo files.

How to use. The top of the page shows the last audit committed to qa/report.json (falls back to qa/report.sample.json) — this is produced by node qa/audit.mjs in CI or locally. Expand any category card to see the offending URLs and severities. Click Run live audit to re-check the deployed pages in your browser right now (broken links, duplicate titles/descriptions, JSON-LD validity, placeholder text, and static accessibility). Live checks are labelled IN-BROWSER; checks that need CI are labelled CI ONLY.
